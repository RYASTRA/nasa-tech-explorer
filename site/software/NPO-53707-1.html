auto-priors-rs - Automatic TCCON priors

This software is specifically designed to handle both ad hoc user requests for input files needed to run GGG for variable times and locations as well as ongoing generation of these files for sites with routinely operating instruments. Compared to other efforts to accomplish this task, auto-priors-rs is significantly more maintainable and extensible. By using Rust as the primary language, the strong type system design minimizes runtime errors. This is important as failures of this system impact users around the globe. It also facilitates future upgrades by allowing more errors to be caught at compile time.
